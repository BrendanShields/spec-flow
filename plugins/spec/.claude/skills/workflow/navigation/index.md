## File Organization Patterns

### The 3-File Pattern
Each of 13 commands has 3 supporting files:

1. **guide.md** - Step-by-step instructions, when to use
2. **examples.md** - Real-world scenarios, before/after
3. **reference.md** - Technical details, API, advanced usage

**Example**: For `/generate` command:
- `phases/2-define/generate/guide.md` - How to generate specs
- `phases/2-define/generate/examples.md` - Example specifications
- `phases/2-define/generate/reference.md` - Spec structure & validation

### Reading Recommendations

**Quick lookup** (2-5 min)
→ Phase README or skill-index.md

**Learning a command** (10-15 min)
→ guide.md → examples.md

**Mastering a command** (20-30 min)
→ guide.md → examples.md → reference.md

**Understanding workflow** (30-60 min)
→ quick-start.md → workflow-map.md → phase READMEs
